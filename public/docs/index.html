Introduction

Base URL: http://localhost

This documentation aims to provide all the information you need to work with our API.

As you scroll, you'll see code examples for working with the API in different programming languages in the dark area to the right (or as part of the content on mobile). You can switch the language used with the tabs at the top right (or from the nav menu at the top left on mobile).

Authenticating requests

This API is not authenticated.

Authentication

Login Api

Example request:
curl --request POST \
    "http://127.0.0.1:8000/api/v1/login" \
    --header "Content-Type: application/json" \
    --header "Accept: application/json" \
    --data "{
    \"email\": \"john@yopmail.com\",
    \"password\": \"12345678\"
}"

const url = new URL(
    "http://127.0.0.1:8000/api/v1/login"
);

const headers = {
    "Content-Type": "application/json",
    "Accept": "application/json",
};

let body = {
    "email": "john@yopmail.com",
    "password": "12345678"
};

fetch(url, {
    method: "POST",
    headers,
    body: JSON.stringify(body),
}).then(response => response.json());

Request      

POST api/v1/doctor/changePassword

Headers

Content-Type      

Example: application/json

Accept      

Example: application/json

Body Parameters

current_password   string   

The value and new_password must be different. The value and confirm_password must be different. Must be at least 8 characters. Example: fzxzhlcqpmrzwlhxryternbhfsgsbmyl

The value and current_password must be different. The value and confirm_password must match. Must be at least 8 characters. Example: piajrbpvckheywgojxrbjavd

confirm_password   string   

The value and current_password must be different. The value and new_password must match. Must be at least 8 characters. Example: nrexplyxhlxcrggumvqpcatknyoynvyfmxmosdtdhtckgfxanjuwutdii

old_password   string   

The old_password of the Doctor. Example: dolores

password   string   

The password of the Doctor. Example: ioqH6\y(g\IOYb"

password_confirmation   string   

The password_confirmation of the Doctor. Example: ea
